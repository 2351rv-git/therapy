# 의료기기 일상점검 관리 대장 웹 애플리케이션

제공해주신 실무 점검 대장 양식을 기반으로 제작된 **의료기기 일상점검 관리 및 인쇄용 웹 애플리케이션**입니다. 별도의 서버 설치 없이 웹 브라우저 자체에서 구동되며, 데이터가 유실되지 않도록 보관 및 인쇄 최적화 기능을 갖추고 있습니다.

## 주요 기능

1. **실무 양식 100% 매칭**:
   - 제목, 부서명 입력란, 공통 점검 항목 가이드라인, 점검 대장 테이블, 그리고 하단 결재란(담당자/파트장/부서장)까지 제공해주신 이미지 양식을 완벽히 재현했습니다.
2. **인쇄 최적화 (A4 가로 Landscape)**:
   - 화면상에 있는 편집용 버튼(테마 토글, 기기 관리, 백업 등)은 인쇄 시 자동으로 제외되며, 표와 서명란만 A4 가로 규격 한 장에 딱 맞추어 선명하게 인쇄됩니다.
3. **편리한 입력 방식**:
   - **클릭 토글**: 빈 셀을 클릭하면 즉시 `V`(적합)로 표시되며, 다시 클릭하면 빈칸으로 돌아갑니다.
   - **상세 입력 (더블클릭 / 우클릭)**: 팝오버 창이 나타나 공통 점검 항목 코드(① 작동상태 ~ ⑤ 청결상태)를 마우스 클릭만으로 입력하거나, 직접 텍스트(예: "점검불가", "휴가")를 적어넣을 수 있습니다.
   - **빠른 일괄 입력**: 오늘 날짜의 모든 장비, 또는 특정 일자의 모든 장비, 특정 장비의 한 달 데이터를 클릭 한 번으로 모두 `V`(적합) 처리할 수 있어 매일 발생하는 대량의 반복 입력을 획기적으로 줄여줍니다.
4. **연/월 자동 대입 및 주말 하이라이트**:
   - 연도와 월을 선택하면 해당 월의 실제 일수(28~31일)에 맞춰 테이블 열이 유동적으로 조절되며, 토요일과 일요일은 눈에 띄게 시각적으로 표시됩니다.
5. **브라우저 로컬 데이터 저장 (LocalStorage)**:
   - 입력한 장비명 목록, 부서명, 담당자명, 그리고 일자별 점검 기록은 브라우저 내부에 연/월별로 안전하게 자동 저장됩니다. 브라우저를 닫거나 새로고침해도 데이터가 보존됩니다.
6. **백업 내보내기 및 가져오기 (JSON)**:
   - 작성한 모든 데이터를 하나의 `.json` 파일로 저장(백업)해 둘 수 있으며, PC를 변경하더라도 해당 파일을 다시 불러와 작업을 이어갈 수 있습니다.
7. **프리미엄 반응형 UI 및 다크 모드**:
   - 주간/야간 업무 환경에 맞추어 라이트/다크 테마를 토글할 수 있는 미려한 UI 디자인이 적용되었습니다.

## 🚀 GitHub & Vercel 배포 방법

본 프로젝트는 데이터베이스나 서버 백엔드가 필요 없는 **순수 정적(Static) 웹사이트**입니다. 따라서 별도의 빌드 과정 없이 깃허브에 올리고 버셀(Vercel)에 연동하면 1초 만에 무료로 배포가 완료됩니다.

### 1단계: Git 저장소 초기화 및 GitHub 업로드

1. 터미널(또는 Git Bash)을 열고 프로젝트 폴더로 이동합니다:
   ```bash
   cd C:\Users\W10asb01\.gemini\antigravity\scratch\equipment-manager
   ```
2. Git 저장소를 생성하고 파일을 추가합니다:
   ```bash
   git init
   git add .
   git commit -m "Initial commit of Equipment Manager Checklist WebApp"
   ```
3. 본인의 GitHub에서 새 레포지토리(예: `medical-equipment-manager`)를 생성합니다.
4. GitHub에 프로젝트를 푸시합니다:
   ```bash
   git branch -M main
   git remote add origin https://github.com/사용자이름/레포지토리이름.git
   git push -u origin main
   ```

### 2단계: Vercel에서 무료 배포하기

1. [Vercel 공식 홈페이지](https://vercel.com/)에 접속하여 가입 후 로그인합니다 (GitHub 계정 연동 권장).
2. 대시보드에서 **[Add New...]** -> **[Project]** 버튼을 클릭합니다.
3. 방금 푸시한 GitHub 레포지토리를 목록에서 찾아 **[Import]**를 누릅니다.
4. **Configure Project** 화면이 나오면 아래와 같이 기본값 그대로 유지합니다:
   - **Framework Preset**: `Other` (자동 감지됨)
   - **Root Directory**: `./` (기본값)
   - **Build and Development Settings**: 설정할 필요 없음 (공란)
5. **[Deploy]** 버튼을 클릭합니다.
6. 약 10~20초 후 배포가 완료되며 Vercel이 제공하는 고유 인터넷 주소(예: `https://your-project.vercel.app`)로 접속하여 즉시 사용할 수 있습니다!
